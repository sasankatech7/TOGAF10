# TOGAF ADM Phase F – Migration Planning

## Overview

Phase F (Migration Planning) converts the implementation opportunities identified in Phase E into a detailed and prioritized Implementation and Migration Plan.

The primary focus of this phase is to determine the order, timing, dependencies, resources, and business priorities for implementing the target architecture.

Phase F answers the question:

> "How and when will we move from the current architecture to the target architecture?"

---

# Purpose

The purpose of Phase F is to:

* Prioritize implementation projects and work packages.
* Create a detailed Implementation and Migration Plan.
* Finalize the Architecture Roadmap.
* Assign business value to implementation initiatives.
* Confirm Transition Architectures.
* Align architecture initiatives with enterprise portfolio management and governance processes.

---

# Objectives

The objectives of Phase F are:

1. Finalize the Architecture Roadmap.
2. Create a practical Implementation and Migration Plan.
3. Prioritize projects based on business value and risk.
4. Validate Transition Architectures.
5. Estimate resources, costs, and schedules.
6. Align architecture initiatives with portfolio and project management.
7. Prepare implementation activities for Phase G.

# Approach

Phase F follows a planning and prioritization approach.

The overall approach is:

1. Review implementation strategy from Phase E.
2. Evaluate business value of work packages.
3. Assess resource requirements.
4. Prioritize projects.
5. Validate Transition Architectures.
6. Develop migration schedules.
7. Finalize Architecture Roadmap.
8. Create Implementation and Migration Plan.
9. Prepare governance inputs for Phase G.

# One-Line Summary

Phase F transforms the solution opportunities identified in Phase E into a prioritized, resource-aware, and executable Implementation and Migration Plan that guides the organization's journey from the baseline architecture to the target architecture.

For TOGAF 10 exam preparation, the **single most important distinction** is:

| Phase E                     | Phase F                                                |
| --------------------------- | ------------------------------------------------------ |
| *What should we implement?* | *How, when, and in what order should we implement it?* |

This distinction appears frequently in Foundation and Practitioner exam scenarios.
