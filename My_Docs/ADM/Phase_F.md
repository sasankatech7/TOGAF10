In TOGAF ADM, **Phase F: Migration Planning** focuses on creating a practical roadmap to move from the **Baseline Architecture** to the **Target Architecture**.

### Objectives of Phase F

* Finalize the Architecture Roadmap.
* Create a detailed Implementation and Migration Plan.
* Prioritize projects and work packages.
* Align migration activities with enterprise portfolio and project management.
* Ensure stakeholders understand costs, risks, and business value. ([coe.qualiware.com][1])

## Steps in ADM Phase F (Migration Planning)

### 1. Confirm Management Framework Interactions

Ensure the migration plan aligns with:

* Business planning
* Portfolio management
* Project management
* Operations management
* Governance processes

This prevents architecture initiatives from conflicting with existing organizational programs. ([coe.qualiware.com][1])

### 2. Assign Business Value to Each Work Package

For every project or work package:

* Identify business benefits.
* Define ROI expectations.
* Evaluate strategic alignment.
* Assess contribution to business goals.

Typical measures include:

* Revenue increase
* Cost reduction
* Risk reduction
* Compliance improvement
* Customer satisfaction improvement. ([Visual Paradigm TOGAF][2])

### 3. Estimate Resources, Timelines, and Delivery Approach

Determine:

* Required budget
* Human resources
* Technology resources
* Project duration
* Delivery vehicle (Agile, Waterfall, Hybrid, etc.)

This produces realistic implementation schedules. ([coe.qualiware.com][1])

### 4. Prioritize Migration Projects

Perform:

* Cost-benefit analysis
* Risk assessment
* Dependency analysis
* Feasibility assessment

Projects with higher value and lower risk are typically scheduled earlier. ([coe.qualiware.com][1])

### 5. Confirm Transition Architectures

Validate intermediate states between current and target architectures.

Example:

```text
Current CRM
      ↓
Transition Architecture 1
      ↓
Transition Architecture 2
      ↓
Target CRM Platform
```

Ensure each transition state is achievable and provides business value. ([Visual Paradigm TOGAF][2])

### 6. Generate the Detailed Architecture Roadmap

Create the implementation sequence:

* Work packages
* Milestones
* Dependencies
* Releases
* Transition architectures

This roadmap becomes the execution guide for transformation initiatives. ([Visual Paradigm TOGAF][2])

### 7. Complete the Architecture Development Cycle and Document Lessons Learned

* Finalize migration planning artifacts.
* Capture lessons learned.
* Update architecture repository.
* Prepare for implementation governance (Phase G).
* Raise new architecture work requests if required. ([coe.qualiware.com][1])

## Easy Exam Mnemonic

**C-A-E-P-C-G-C**

**C** – Confirm management framework interactions
**A** – Assign business value
**E** – Estimate resources and timings
**P** – Prioritize migration projects
**C** – Confirm transition architectures
**G** – Generate architecture roadmap
**C** – Complete cycle & lessons learned

## Key Deliverables of Phase F

* Implementation & Migration Plan
* Finalized Architecture Roadmap
* Updated Architecture Definition Document
* Updated Architecture Requirements Specification
* Transition Architectures
* Implementation Governance Model
* Requests for Architecture Work (next iteration) ([coe.qualiware.com][1])

### ADM Flow Context

```text
Phase E → Opportunities & Solutions
            ↓
Phase F → Migration Planning
            ↓
Phase G → Implementation Governance
            ↓
Phase H → Architecture Change Management
```

For TOGAF certification exams, remember that **Phase E identifies solution options and work packages**, while **Phase F converts those options into a prioritized, executable migration roadmap and implementation plan**. ([circle.visual-paradigm.com][3])

### Exam Keyword Chain

**CAVEPRG**

Pronounce it as:

> **"CAVE PRG"**

* **C** = Confirm Frameworks
* **A** = Assign Value
* **V** = Value Assessment
* **E** = Estimate Resources
* **P** = Prioritize Projects
* **R** = Roadmap Generation
* **G** = Governance Preparation

Or the more accurate sequence:

**CAEPCGC**

* Confirm
* Assign Value
* Estimate
* Prioritize
* Confirm Transition Architectures
* Generate Roadmap
* Complete Cycle

### One-Line Exam Summary

> **Phase E decides WHAT solutions to implement; Phase F decides WHEN, IN WHAT ORDER, and WITH WHAT RESOURCES they will be implemented.**

That single sentence helps distinguish Phase F from Phase E in many TOGAF 10 exam questions.
